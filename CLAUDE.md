# Claude Code 프로젝트 지시사항

## 구현 전 확인 (최우선 준수)

**코드 구현을 시작하기 전에 반드시:**

1. 구현 플랜과 구현 내용을 간략하게 브리핑
2. 사용자에게 구현 진행 여부 확인을 받은 후 작업 시작

> 확인 없이 임의로 구현하지 말 것.

## 프로젝트 개요

game-devcode-kr - WebGL 기반 브라우저 게임 프로젝트

## 기술 스택

- **게임 엔진:** Phaser 3 (WebGL 2 / Canvas 2D)
- **프레임워크:** Next.js 15+ (App Router)
- **언어:** TypeScript
- **서버:** Next.js API Routes (리더보드, 세이브 데이터)
- **DB:** Supabase 또는 Vercel Postgres (추후 확정)
- **배포:** Vercel

> 상세 내용: `/docs/tech-stack.md` 참고

## 개발 규칙

### 코드 작성 시 준수사항

1. **정보 부족 시 질문**: 구현에 필요한 정보가 부족하면 임의로 구현하지 말고 반드시 추가 정보를 요청할 것
2. **작업 문서화**: 모든 작업 내용은 `/docs/tasks/` 폴더 아래에 브랜치명으로 마크다운 파일 생성하여 기록
3. **한국어 우선**: 문서 및 커밋 메시지는 한국어로 작성

### Git 워크플로우 (필수)

**모든 코드 변경은 반드시 별도 브랜치에서 작업하고 Pull Request를 통해 병합합니다.**

1. **브랜치 생성**: 작업 전 반드시 새 브랜치 생성
2. **작업 진행**: 해당 브랜치에서 개발
3. **커밋/푸시 전 확인**: 커밋/푸시 전에 반드시 사용자 허락을 받을 것
4. **푸시**: 원격 저장소에 브랜치 푸시
5. **PR 생성**: GitHub에서 Pull Request 생성
6. **병합**: 리뷰 후 main 브랜치에 병합

> ⚠️ **main 브랜치에 직접 커밋/푸시 금지**

### 브랜치 네이밍

- **새 기능 추가**: `feat/기능명` (예: `feat/player-movement`)
- **오류 수정**: `fix/수정내용` (예: `fix/collision-bug`)
- **핫픽스**: `hotfix/수정내용`
- **리팩토링**: `refactor/내용`
- **문서 수정**: `docs/내용` (예: `docs/engine-research`)

### 커밋 메시지

- 한국어로 작성
- 형식: `[타입] 내용`
- 타입: feat, fix, docs, style, refactor, test, chore
