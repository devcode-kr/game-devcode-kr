# BM Advertising Notes

## 목적
- 게임 안에 광고를 노골적인 배너 형태로 넣기보다, 월드와 상호작용 구조 안에 자연스럽게 섞이는 BM 아이디어를 정리한다.
- 실제 구현은 나중에 하되, 광고주용 운영 흐름과 게임 내 노출 방식의 큰 틀을 먼저 기록한다.

## 1. 광고 구좌 기반 텍스쳐 광고

### 개념
- 광고주가 별도 광고주용 사이트에서 특정 광고 구좌를 구매한다.
- 구매한 구좌에 대응하는 게임 내 월드 슬롯에 광고주가 업로드한 이미지를 텍스쳐로 반영한다.
- 노출은 UI 배너가 아니라 월드 텍스쳐 치환 방식으로 처리한다.

### 예시
- 바닥 타일 광고 구좌
  - 광고주가 일정 구간의 바닥 광고 슬롯을 구매
  - 해당 구간마다 일부 바닥 타일 텍스쳐를 광고 이미지 기반 텍스쳐로 교체
- 벽 텍스쳐 광고 구좌
  - 광고주가 벽면 광고 슬롯을 구매
  - 일부 벽 타일이 브랜드 이미지가 들어간 벽 텍스쳐로 표시

### 의도
- 배너 광고처럼 몰입을 끊지 않음
- 게임 내 배경/환경 일부처럼 보이도록 자연스럽게 섞을 수 있음
- 타일 기반 게임 구조와 잘 맞음

### 운영 측 아이디어
- 광고주용 별도 사이트 필요
- 사이트에서 구좌 목록, 노출 위치 미리보기, 업로드 가이드 제공
- 이미지 업로드 후 검수/승인 절차 필요
- 텍스쳐 규격, 용량, 비율, 안전영역 규칙 필요

### 게임 측 고려 사항
- 광고 텍스쳐는 특정 구간 또는 특정 층/맵 시드와 연결될 수 있음
- 월드 분위기를 깨지 않도록 색감/스타일 필터를 거친 변환 텍스쳐가 필요할 수 있음
- 지나치게 반복 노출되지 않도록 배치 밀도 제한 필요

## 2. 업적 광고

### 개념
- 광고주가 자신만의 업적 조건을 설정한다.
- 플레이어가 해당 업적을 달성하면 광고 관련 NPC가 새로 출몰하거나 기존 NPC 대화가 열림
- NPC와 대화하면 할인 코드, 쿠폰, 제품 안내 등 실제 보상으로 연결할 수 있다.

### 예시
- 광고주 업적: 일반 상자 5개 개봉
- 광고주 업적: 잠긴 상자 3개 개봉
- 광고주 업적: 특정 타입 몬스터 10마리 처치
- 광고주 업적: 3층 도달

### 게임 내 노출 방식
- 숨겨진 업적이면 안 됨
- 플레이어가 미리 목표를 알 수 있어야 하므로 업적 보드에 명시적으로 표시
- 달성 시 새 NPC 출몰 또는 기존 NPC의 새 대사/보상 해금

### 보상 방식 아이디어
- 특정 상품 할인 코드 제공
- 광고주 브랜드 테마 코스메틱 보상
- 외부 사이트 이동용 코드 또는 링크 제공
- 제한 기간 프로모션 이벤트 연동

### 운영 측 고려 사항
- 업적 조건은 시스템적으로 판별 가능한 이벤트여야 함
- 할인/쿠폰은 악용 방지를 위해 발급 이력과 만료 정책이 필요
- 업적 달성 시 광고성 보상이라는 점을 명확히 표시해야 함

## 시스템 관점 정리

### 광고주용 사이트
- 광고 구좌 구매
- 광고 이미지 업로드
- 업적 광고 캠페인 설정
- 노출 기간, 예산, 검수 상태 확인

### 게임 클라이언트
- 광고 슬롯에 대응하는 텍스쳐 교체
- 광고 업적 보드 노출
- 업적 달성 시 NPC/보상 해금

### 백엔드/운영
- 광고 소재 저장소
- 캠페인/구좌 관리
- 업적 조건 매핑
- 보상 발급 및 사용 추적

## 리스크 메모
- 게임 분위기와 광고 이미지 스타일 충돌 가능성
- 광고가 과도하면 몰입 저하 가능성
- 할인/쿠폰형 보상은 어뷰징 방지 설계 필요
- 지역/연령/플랫폼별 광고 정책 검토 필요
